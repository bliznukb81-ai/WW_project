Адаптивная мини-витрина магазина, взаимодействующая с публичным API.

Онлайн версия: https://wwproject.netlify.app/

## Стек технологий
* React 18 (сборщик Vite)
* Tailwind CSS (стилизация и адаптив)
* Zustand (глобальный стейт-менеджмент для корзины и юзера)
* Lucide React (иконки)

## Как запустить проект локально

1. Склонируйте репозиторий:
   git clone https://github.com/bliznukb81-ai/WW_project.git

2. Перейдите в директорию проекта:
   cd WW_project

3. Установите зависимости:
   npm install

4. Настройте переменные окружения:
   * В корне проекта найдите файл .env.example.
   * Создайте рядом с ним файл .env.
   * Вставьте в .env ваш персональный API-ключ:
     VITE_WW_API_KEY=ваш_ключ_здесь

5. Запустите сервер для разработки:
   npm run dev

## Что было реализовано (Особенности и UI/UX детали)

Помимо базовых требований из ТЗ (вывод товаров, пагинация, фильтрация), я уделил особое внимание деталям интерфейса и пользовательскому опыту:

* Адаптивный поиск:
  - Реализовано раздельное поведение для разных экранов. На десктопе строка поиска плавно выезжает влево. На мобильных устройствах — аккуратно появляется снизу, не ломая верстку шапки и не перекрывая логотип.
  - Добавлен Debounce (400ms) на ввод текста, чтобы не спамить API лишними запросами при быстрой печати.
* Фиксы для мобильных устройств (iOS):
  - В index.html и стилях инпутов (размер шрифта 16px / text-base) прописана защита от принудительного автоматического зума (Auto-Zoom) при фокусе на полях ввода в Safari на iPhone.
* Умная логика авторизации и корзины:
  - Разделены триггеры входа. Если пользователь нажимает на "Личный кабинет", он просто авторизуется. Если он нажимает "Оформить заказ" в корзине будучи неавторизованным — система запоминает его намерение, открывает модалку входа и только после успешного входа автоматически оформляет заказ и очищает корзину.
* Микроинтеракции и визуал:
  - Sticky Header: "Липкая" шапка с эффектом backdrop-blur, чтобы навигация и доступ к корзине всегда были под рукой при скролле длинного каталога.
  - Бейдж корзины: Анимированный счетчик общего количества товаров на иконке корзины.
  - Skeleton Loading: При загрузке и подгрузке товаров отрисовываются скелетоны, чтобы избежать резких скачков контента (Layout Shifts).
  - Товары со статусом "Нет в наличии" визуально блокируются для клика.

## Что можно улучшить (в рамках реального продакшена)
1. Кэширование: Сейчас данные загружаются через стандартный useEffect. В боевом проекте я бы внедрил TanStack Query (React Query) для кэширования ответов, ревалидации в фоне и более изящной обработки состояний isLoading/isError.
2. Rate Limit: Заголовки X-RateLimit-* сейчас не обрабатываются строго (хватает текущих 60 запросов/мин для демонстрации). В проде добавил бы axios-interceptor для глобального отлова HTTP 429 и показа красивого Toast-уведомления пользователю ("Слишком много запросов, подождите").
3. Роутинг: Для полноценного магазина имеет смысл прикрутить React Router (или перенести на Next.js), чтобы каждый товар имел свой уникальный URL для возможности поделиться ссылкой.
